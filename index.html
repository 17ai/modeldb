ModelDB: a system for managing machine learning models

Project Overview

ModelDB is an end-to-end system to manage machine learning models.

ModelDB is made up of three main parts: the ModelDB server, the ModelDB clients, and the ModelDB frontend.

ModelDB server

The ModelDB server stores all the data for ModelDB and exposes a thrift API for storing as well as querying data.

ModelDB client

ModelDB clients are native libraries in various languages that can be used to log data to ModelDB. We currently provide clients for spark.ml and scikit-learn.

Sample machine learning workflows using these client libraries are available here, here, and here.

ModelDB Frontend

Once machine learning workflows and models have been logged to ModelDB, they can be explored via the web-based ModelDB frontend.

Setup

To setup ModelDB, follow these steps:

  1. ModelDB has various dependencies. See Required Software. Install the relevant ones.
  2. Configure and start the server.
  3. Build modeldb clients as described in for spark.ml and scikit-learn.
  4. Update your ML code to use the client libraries to log workflows and models to ModelDB, as in here, here, and here.
  5. Run your ML code normally.
  6. Start the ModelDB frontend and explore models in ModelDB.
